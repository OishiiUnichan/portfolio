# 01_要件定義

## 1.1. バックエンド要件（Java）

1. **タスク管理機能:** タスクのCRUD操作（登録、一覧取得、削除、完了ステータス更新）ができること。
2. **ガチャトリガーシステム:** タスク完了ごとにカウンターを `+1` し、規定数に達したらガチャ実行権限を付与すること。
* **通常モード:** 3個クリアで1回ガチャ実行。
* **デモモード:** 1個クリアで1回ガチャ実行。


3. **ルーム状態管理:** 各エリアに配置されている家具データ、およびガチャプール（未獲得デコ家具、未カンストコア家具）の差分を管理・保持すること。
4. **アバター状態演算:** システム時刻（hour）を元に、アバターの行動ID（座る、昼寝、散歩など）を判定するロジックを持つこと。

## 1.2. フロントエンド要件（HTML/CSS/JavaScript）

1. **タスク管理UI:** シンプルで実用的なToDoリスト画面であること。
2. **ルーム画面描画:** CSS Grid等を使用し、アイソメトリックな部屋をクォータービューで描画すること。Javaから受け取った配置データ（JSON）をベースに、アセット画像をエリアごとにレイヤー重ね合わせ処理（固定自動配置）すること。
3. **アバター・アニメーション制御:** 指定された行動IDに応じて、アバターのループアニメーション画像を切り替え表示すること。

## 1.3. グラフィック・アセット要件（Blender）

1. **部屋のベース仕様:** 角度を固定したアイソメトリック空間。初期状態として、最低限の基本家具（机、PC、椅子、ベッド、ライト）が配置されていること。
2. **アセットの2大分類:**
* **コア家具 (Core):** 初期配置済みの基本家具。ガチャで重複当選するとレベルが上がり、グラフィックが進化する。
* **デコ家具 (Deco):** タスククリアで新しく追加される小物（マグカップ、植物など）。当選すると指定エリアに自動で増えていく（1スロット複数同居型）。

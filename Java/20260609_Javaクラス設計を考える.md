# クラス設計メモ

## 基本情報

| 項目 | 内容 |
|------|------|
| アプリ名 | movielog |
| パッケージ名 | `com.example.movielog` |

## 必要なクラス一覧

| クラス名 | 役割 |
|----------|------|
| `Movie` | DBのテーブルをJavaで表現（データの入れ物） |
| `MovieRepository` | DBとのやりとり（Spring Bootが自動生成） |
| `MovieService` | 処理の流れ制御、入力値チェック |
| `MovieController` | ボタンを押したときの処理（画面とのやりとり） |

## 作成順番

依存が少ない順に作る。

1. `Movie`（他に依存しない）
2. `MovieRepository`（Movieに依存）
3. `MovieService`（MovieとMovieRepositoryに依存）
4. `MovieController`（全部に依存）

## 全体の流れ

```
MovieController（画面からの操作を受け取る）
　↓
MovieService（入力値チェック・処理の制御）
　↓
MovieRepository（DBとのやりとり）
　↓
DB
```

## 各クラスの詳細

### Movie
DBのテーブル構造に対応したフィールドを持つ。  
Spring Bootではアノテーションを追加する。

```java
int id
LocalDate watchDate
String title
String media
String genre
```

### MovieRepository
Spring Bootが自動でSQLを生成してくれる。  
自分でSQL文を書く必要がほぼない。

```java
save(Movie movie)    // 登録・更新
findAll()            // 一覧取得
deleteById(int id)   // 削除
```

### MovieService
入力値のチェックとMovieRepositoryの呼び出しを行う。

### MovieController
画面からのリクエストを受け取り、MovieServiceを呼び出す。  
HTMLと連携する。

## 昔の設計との変更点

| 昔のクラス | Spring Bootでの対応 |
|------------|-------------------|
| `Main` | Spring Bootが自動でやってくれる部分が増える |
| `Movie` | ほぼそのまま、アノテーション追加 |
| `MovieDAO` | `MovieRepository`に変更（SQLを自動生成） |
| `Service` | `MovieService`にほぼそのまま |
| `Input` | `MovieController` + HTMLに分離 |

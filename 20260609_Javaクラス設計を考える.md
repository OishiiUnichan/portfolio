# クラス設計

## 考え方：役割で分ける

クラス設計は「このアプリで何をするか」を役割ごとに分類して考える。

---

## 必要なクラス一覧

| クラス名 | 役割 |
|----------|------|
| `Main` | 起動（mainメソッドを持つ） |
| `Input` | 画面・入力（ユーザーからの入力を受け取る） |
| `Service` | 処理の流れ制御（入力値のチェック、DAOの呼び出し） |
| `MovieDAO` | DBの操作（SQL文を発行してDBとやりとりする） |
| `Movie` | データの入れ物（DBの1レコードをJavaのオブジェクトとして持ち運ぶ） |

---

## 全体の流れ

```
Main
　↓
Input（画面・入力）
　↓
Service（処理の流れ制御）
　↓
MovieDAO（SQL発行）
　↓
DB
```

---

## 各クラスの詳細

### Movie
DBのテーブル構造に対応したフィールドを持つ。

```java
int id
LocalDate watchDate
String title
String genre
```

### MovieDAO
SQL文を書く場所はここだけ！というルールにする。  
DAO=Data Access Object

```java
insert(Movie movie)    // 登録
findAll()              // 一覧取得
update(Movie movie)    // 更新
delete(int id)         // 削除
```

### Service
入力値のチェックとDAOの呼び出しを行う。

```java
// 処理の流れ
入力値を受け取る
　↓
入力値のチェック
　↓
DAOのメソッドを呼び出す
```

### Input
ユーザーからの入力を受け取り、Serviceに渡す。

### Main
アプリの起動点。Inputを呼び出す。

---

### 思考ログ
１番小さい部品から作っていく。１番小さい部品＝他のクラスに依存しないもの。
Movie→MovieDAO→Service→Input→Main
